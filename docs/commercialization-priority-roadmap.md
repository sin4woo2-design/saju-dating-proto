# 사주앱 상용화 우선순위 로드맵 (v1)

기준: **매출 영향 × 리스크 감소 × 구현 난이도(낮을수록 우선)**
작성일: 2026-03-14 (UTC)

## P0 (이번 주, 즉시 진행)

### P0-01. 이벤트 스키마 + 퍼널 대시보드 정의
- 목표: 온보딩→결과조회→결제 전환 구간을 수치로 추적
- 산출물
  - 이벤트 명세서 (`docs/analytics-event-schema.md`)
  - 핵심 KPI 정의 (온보딩 완료율, 결제 진입률, 결제 성공률, D1/D7 리텐션)
- 완료 기준
  - 이벤트 이름/파라미터/트리거 지점 확정
  - 대시보드에서 퍼널 1개 이상 조회 가능

### P0-02. 결제 퍼널 안정화 체크리스트
- 목표: 결제 실패/복구 누락 최소화
- 산출물
  - 결제 시나리오 테스트 목록 (`docs/payment-funnel-qa-checklist.md`)
  - 실패/복구 UX 문구 초안
- 완료 기준
  - 신규결제/복구/실패재시도/취소 케이스 수동 테스트 가능

### P0-03. 온보딩 3-step 단순화 설계
- 목표: 첫 결과 도달 시간 단축(이탈 감소)
- 산출물
  - 온보딩 단계 축소안 (`docs/onboarding-optimization-plan.md`)
  - 필수 입력/선택 입력 분리안
- 완료 기준
  - 첫 결과까지 단계 수, 입력 개수, 예상 소요시간 명시

## P1 (1~2주)

### P1-01. 결과 리포트 상품 구조 개선
- 무료 요약 + 유료 심화 리포트 구조 고도화
- 공유 카드(한 줄 요약) 설계

### P1-02. 리텐션 장치 도입
- 데일리 운세 알림, 궁합 재조회 유도
- 주간 리캡 카드 실험

### P1-03. 리뷰/CS 대응 체계
- 문의 채널/FAQ/응답 템플릿
- 평점 하락 키워드 트래킹

---

## P2 (2~4주)

### P2-01. 법무/정책 패키지
- 개인정보처리방침/이용약관/환불정책/면책문구

### P2-02. 품질 모니터링
- Crashlytics/Sentry 도입
- 알림 룰(치명 오류, 결제 실패 급증)

### P2-03. A/B 테스트 루프
- 페이월 문구/가격/플랜 실험
- 온보딩 카피 실험

---

## 벤치마킹 운영 방식

### 1차 비교군 (필수)
- 결제 퍼널 강한 앱 5개
- 온보딩 전환 강한 앱 5개

### 2차 비교군 (선택)
- 리포트 몰입도 높은 앱 3개
- 리텐션이 강한 앱 3개

### 평가 항목 (공통)
- 온보딩 단계/입력량/첫결과 도달시간
- 무료/유료 경계(언제 과금 유도하는지)
- 결제 옵션/가격/복구 UX
- 결과물 신뢰감(카피, 설명 구조, 시각화)
- 재방문 장치(알림, 리마인드, 콘텐츠 루프)
- 리뷰 불만 키워드

1. 이벤트 명세 확정
2. 결제/온보딩 실험
3. 리포트 상품성 개선
4. 리텐션 루프 삽입
5. 법무/품질/AB 인프라 고도화
